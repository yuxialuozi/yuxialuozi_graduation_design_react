## 二、课题背景及意义

### 2.1 研究背景

随着城市化进程的加速和人口流动性的增加，房屋租赁市场规模不断扩大。传统的房屋租赁管理方式主要依赖纸质档案和人工记录，存在以下问题：

1. **信息管理混乱**：租户信息、合同信息、费用记录分散管理，难以统一查询和统计
2. **工作效率低下**：手工录入和查找信息耗时耗力，容易出现人为错误
3. **数据安全隐患**：纸质档案易丢失、损坏，缺乏有效的数据备份机制
4. **统计分析困难**：无法快速生成经营报表，难以进行数据驱动的决策
5. **服务响应滞后**：维修工单处理流程不透明，租户体验差

<!-- 截图位置：现有房屋租赁市场问题分析图 - 可绘制问题分析思维导图或痛点分析图表 -->

### 2.2 研究意义

本课题设计并实现一套现代化的房屋出租管理系统，具有以下意义：

| 维度 | 意义说明 |
|------|----------|
| **理论意义** | 探索前后端分离架构在房产管理领域的应用实践，为同类系统开发提供参考 |
| **实践意义** | 提高房屋租赁管理效率，降低运营成本，提升租户满意度 |
| **技术意义** | 深入实践 React + TypeScript 前端技术栈与 Go 微服务后端技术栈 |
| **社会意义** | 推动房屋租赁行业信息化、数字化转型 |

// 租户信息

  id: number;
  name: string;
  contactPerson: string;
  phone: string;
  email: string;
  status: 'active' | 'inactive';
  createdAt: string;
  updatedAt: string;

interface Contract {
  id: number;
  tenantId: number;
  tenantName: string;
  contractNo: string;
  startDate: string;
  endDate: string;
  amount: number;
  status: 'draft' | 'active' | 'expired' | 'terminated';
}

// 房间信息
interface Room {
  id: number;
  roomNo: string;
  building: string;
  floor: number;
  area: number;
  monthlyRent: number;
  status: 'vacant' | 'occupied' | 'maintenance';
  tenantId?: number;
  tenantName?: string;
}

// 费用记录
interface Fee {
  id: number;
  tenantId: number;
  roomNo: string;
  feeType: 'rent' | 'water' | 'electricity' | 'property' | 'other';
  amount: number;
  period: string;
  dueDate: string;
  paidDate?: string;
  status: 'unpaid' | 'paid' | 'overdue';
}

// 维修工单
interface Maintenance {
  id: number;
  ticketNo: string;
  tenantId: number;
  roomNo: string;
  type: 'electrical' | 'plumbing' | 'appliance' | 'furniture' | 'other';
  description: string;
  priority: 'low' | 'medium' | 'high' | 'urgent';
  status: 'pending' | 'processing' | 'completed' | 'cancelled';
